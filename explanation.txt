Исходный запрос:
SELECT full_name, birth_date, gender 
FROM employees 
WHERE gender = 'Male' AND full_name LIKE 'F%'

Проблемы:
1. Проверка двух условий для каждой строки
2. Нет использования индексов

Результат без оптимизации:
- Время выполнения: 100-500 мс (зависит от оборудования)
- Ресурсоемко: загружает всю таблицу в память

Решение - создание составного индекса:
CREATE INDEX idx_gender_lastname ON employees(gender, full_name)

Почему это работает:
1. Дерево индекса сначала фильтрует по gender = 'Male'
2. Затем ищет full_name LIKE 'F%' только в мужских записях
3. СУБД получает прямые указатели на нужные строки

Технические детали:
1. Индекс хранит значения полей в отсортированном виде
2. Для gender используется бинарный поиск (быстро)
3. Для префикса фамилии (F%) - поиск по диапазону

Результаты после оптимизации:
- Использует только индекс, не загружает всю таблицу
